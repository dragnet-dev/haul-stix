# haul-stix

STIX 2.1 bundles generated by [Dragnet](https://github.com/dragnet-dev/dragnet) and pushed
here every 6 hours.

This repo is a **derivative** of [`dragnet-dev/haul`](https://github.com/dragnet-dev/haul) — it has no engine of its
own. The haul workflow runs sync + generate, then pushes the STIX bundles
into here while pushing the underlying incident data to `haul`.

## Layout

```
feeds/stix/
└── bundle.json                 # combined cross-module bundle

{module}/feeds/stix/
└── bundle.json                 # per-module bundle (one per module)
```

Modules: `supply`, `malware`, `ransomware`, `cve`, `container`.

Large bundles are sharded as `bundle-0.json`, `bundle-1.json`, etc. to stay
under GitHub's 100 MB hard limit per file. Each shard is itself a valid
standalone STIX 2.1 bundle, so consumers can fetch one or all.

## Consuming the bundles

### MISP, OpenCTI, ThreatConnect, custom TIP

Subscribe to the raw URL as a STIX feed:

```
https://raw.githubusercontent.com/dragnet-dev/haul-stix/main/feeds/stix/bundle.json
```

Or per-module if you only want supply chain data:

```
https://raw.githubusercontent.com/dragnet-dev/haul-stix/main/supply/feeds/stix/bundle.json
```

### Cross-references back to incidents

Every SDO/SCO in the bundle includes an `external_references` entry with
`source_name: "dragnet"` and `external_id` set to the canonical incident ID
(`dragnet-{module}-{year}-{seq}`). To fetch the full incident metadata
behind any bundled object, hit haul:

```
https://raw.githubusercontent.com/dragnet-dev/haul/main/{module}/incidents/all/{shard}.jsonl
```

### Full inventory

Start at the orchestrator file in haul:

```
https://raw.githubusercontent.com/dragnet-dev/haul/main/index.json
```

It points at every satellite (rules + STIX) plus the manifest, which carries
sha256s of every file (including bundles in this repo) for cache invalidation.

## Update cadence

Every 6 hours, driven by the haul workflow. Bundles are regenerated from the
current curated incident set each cycle — there's no incremental shape, the
whole bundle reflects the latest state.

## Issues + contributions

This repo is auto-generated. **Don't open PRs against bundle files directly** —
they'll be overwritten on the next sync. Instead:

- Bug in a generated bundle? Open an issue in [`dragnet`](https://github.com/dragnet-dev/dragnet) (the engine).
- Want richer SDO metadata? Open a feature request in `dragnet`.
